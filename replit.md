# Workspace

## Overview

pnpm workspace monorepo using TypeScript. Label Studio ‚Äî a label management application for a scented products business (soy candles, room sprays, room diffusers). Manages the full label production pipeline: brand settings ‚Üí label sheet templates ‚Üí label template designer ‚Üí print queue.

## Stack

- **Monorepo tool**: pnpm workspaces
- **Node.js version**: 24
- **Package manager**: pnpm
- **TypeScript version**: 5.9 (strict-leaning individual flags; `composite: true` on `lib/db`, `lib/api-client-react`, `lib/api-zod` only ‚Äî not on `lib/api-spec` or `artifacts/*`)
- **API framework**: Express 5
- **Database**: PostgreSQL + Drizzle ORM
- **Validation**: Zod (`zod/v4`), `drizzle-zod`
- **API codegen**: Orval (from OpenAPI 3.1 spec)
- **Build**: esbuild (via `build.mjs`)
- **Frontend**: React 19 + Vite + Tailwind CSS + shadcn/ui + wouter
- **Data fetching**: TanStack Query v5 (hooks generated by Orval)
- **PDF generation**: manual low-level PDF assembly (no library ‚Äî raw PDF objects + xref table)
- **File uploads**: multer (memory storage)
- **Streaming**: Server-Sent Events (SSE) for PDF analysis progress

## Database Schema

Tables (all in `lib/db/src/schema/`):
- `label_sheets` ‚Äî OnlineLabels sheet templates with physical dimensions, corner radius, safe area/bleed. OL-only (no Avery). Measurements extracted from PDF vector paths.
- `label_templates` ‚Äî Label layout zone templates (JSONB zones with percentage-based positioning). Associated with a label sheet. Supports per-template safe area guides. `parent_template_id` FK enables 3-level inheritance hierarchy (base type ‚Üí size variant ‚Üí product-specific).
- `design_system` ‚Äî Brand settings singleton (colors, fonts, logo URL, business info). One row, always `id = 1`.
- `products` ‚Äî Product catalog (4 types: soy_candle / room_spray / room_diffuser / other). Required fields: `name`, `scent_name`, `size`. Optional FK to `label_templates`.
- `print_jobs` ‚Äî Print job queue with JSONB items array `[{productId, quantity}]`, status lifecycle.

## API Endpoints

All routes prefixed with `/api`:
- `GET/POST /label-sheets` ‚Äî Label sheet CRUD
- `GET/PATCH/DELETE /label-sheets/:id`
- `GET /label-sheets/:id/pdf` ‚Äî Generate two-layer printable PDF (OCG "Label Borders" + "Guides")
- `POST /label-sheets/upload-pdf` ‚Äî Upload PDF templates (multer memory storage, returns single `jobId` + `fileCount` + `filenames[]` for the batch)
- `GET /label-sheets/analyze/:jobId/events` ‚Äî SSE stream of 12-step analysis progress
- `POST /label-sheets/import` ‚Äî Bulk-import extracted measurements to DB
- `GET/POST /label-templates` ‚Äî Label template CRUD
- `GET/PATCH/DELETE /label-templates/:id`
- `POST /label-templates/:id/cascade` ‚Äî Cascade parent zone layout to all descendant templates (recursive; skips zones with `inheritedFromParent: false`)
- `GET/PATCH /design-system` ‚Äî Brand design system (singleton; GET seeds defaults if empty)
- `GET/POST /products` ‚Äî Product catalog (supports `?productType=` and `?search=` filters)
- `GET/PATCH/DELETE /products/:id`
- `GET/POST /print-jobs` ‚Äî Print job queue
- `GET/PATCH/DELETE /print-jobs/:id`
- `GET /dashboard/stats` ‚Äî Summary stats (sheet/template/product/job counts)
- `GET /dashboard/recent-print-jobs` ‚Äî 5 most recent print jobs
- `GET /dashboard/products-by-type` ‚Äî Product count by type

## PDF Template Analysis

- Measurements extracted from actual PDF vector drawing commands using `zlib.inflateSync`
- Two PDF modes: `re` operator (square corners) and Bezier paths (rounded corners)
- Y-flip coordinate transform (`1 0 0 -1 0 H cm`) auto-detected and handled
- 12-step checklist streamed via SSE per file; H/V math validated (tolerance ¬±0.02")
- Reference: `docs/LABEL_SHEET_MEASUREMENT_PROTOCOL.md`

## PDF Download (Sheet Border PDF)

- Manual low-level PDF assembly (raw PDF objects + xref table; no library dependency)
- Both OCG layers draw the **same** label die-cut outlines (rectangle `re` or Bezier paths):
  - OCG "Label Borders" (black, ~1pt) ‚Äî printable
  - OCG "Guides" (Illustrator guide blue, ~0.5pt) ‚Äî display-only, excluded from print via `AS` event
- Margin/safe-area/bleed lines are NOT yet in the PDF output (browser preview only)
- Rounded-corner labels use cubic Bezier approximation (k = 0.5523)
- Route: `GET /api/label-sheets/:id/pdf`

## Typography & Kerning

- Global CSS `font-feature-settings: "kern" 1` on `body` enables font kerning table
- `.kern-on` / `.kern-off` utility classes available to override per-element
- Brand Settings page shows side-by-side kern-on vs. kern-off for AV, AW, LT, TA
- Font upload cards use browser `FontFace` API for session-local custom font loading
- Accepted font formats: .ttf, .otf, .woff, .woff2

## Label Sheet Form ‚Äî Corner Radius

- The Add/Edit sheet form includes a "Corner Radius (in)" input
- Only enabled when shape = "rectangle"; auto-cleared when switching to circle/oval
- Shows live mm conversion (e.g. 0.125" = 3.18 mm) as helper text
- Stored as `cornerRadius` (real, nullable) in the DB; null = square corners
- Shape dropdown labels: "Rectangle", "Circle", "Oval (ellipse die-cut)" with clarifying note

## Safe Area & Bleed

- Sheet form has collapsible "Print Safe Area & Bleed" section with toggle switch
- Template form has per-template safe area toggle

## TypeScript & Composite Projects

Every package extends `tsconfig.base.json`. Three library packages (`lib/db`, `lib/api-client-react`, `lib/api-zod`) add `composite: true`. `lib/api-spec` has no tsconfig. `artifacts/api-server` and `artifacts/label-studio` use project `references`; `mockup-sandbox` and `scripts` do not. Run typechecks from root:
- `pnpm run typecheck` ‚Äî full workspace typecheck
- `pnpm --filter @workspace/api-spec run codegen` ‚Äî regenerate API client and Zod schemas from OpenAPI spec
- `pnpm --filter @workspace/db run push` ‚Äî push Drizzle schema to DB
- `pnpm --filter @workspace/scripts run seed` ‚Äî seed the database

## Shell & Layout Architecture (UX Overhaul)

- **Custom icon sidebar** (`app-sidebar.tsx`): 56px wide, icon-only at rest; hovering a nav item reveals a flyout tooltip label to the right (CSS `group-hover/nav` + absolute positioned popover). No hamburger/toggle button.
- **Nav order**: Dashboard ‚Üí Print Jobs ‚Üí Products ‚Üí Zones ‚Üí Designs ‚Üí Sheets ‚Üí Branding
- **Branding** is in the main nav (previously called "Brand Settings" in a footer)
- **ShellContext** (`src/context/shell-context.tsx`): pages inject `{ title, actions }` into the topbar via `useShell()` + `useEffect`. Shell reads `topBarState` to render the title and right-side action slot.
- **Shell main**: `overflow-hidden` ‚Äî no padding; pages manage their own layout.
- **PageWrapper** (`src/components/layout/page-wrapper.tsx`): standard scrollable padded layout for all regular pages (Dashboard, Products, Sheets, Print Jobs, Branding). Use `<PageWrapper>` as the root return element.
- **Designs page**: full-bleed editor layout ‚Äî left panel (`w-52 border-r overflow-y-auto`) and right panel (`w-52 border-l overflow-y-auto`) dock flush to edges; canvas fills center. Save button + sheet selector injected into topbar via ShellContext. Empty state replaces always-on header.
- **Zones page**: full-bleed split panel (`h-full`), with a compact action bar at top (border-b, px-4 py-2). Uses `h-full`, not `h-[calc(100vh-Npx)]`.

## Important Conventions

- **API import paths**: always use `@workspace/db` (not `@workspace/db/schema`) for `db` and table exports
- **Frontend API calls**: always use relative `/api/...` paths ‚Äî Replit routes them to the API server
- **Generated files**: never hand-edit `lib/api-client-react/` or `lib/api-zod/`
- **Measurement source of truth**: PDF vector paths, not web pages or catalog descriptions
- **DB single-row tables**: `design_system` always has `id = 1`; GET endpoint seeds defaults if empty
